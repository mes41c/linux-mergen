[PREREQUISITES] REQUIREMENTS
Mergen is a hybrid system running between your computer (Linux) and your phone 
(Android). For full capacity usage, you will need:

1. Google Gemini API Key (Optional - For AI Analysis):
   Go to https://aistudio.google.com/app/apikey and click "Create API Key".
2. Syncthing: To sync the database between two devices.
3. Tailscale (Optional): For remote access (SSH).

--------------------------------------------------------------------------------

[1] INSTALLATION (STEP BY STEP)

A. HOST MACHINE (KALI/UBUNTU)
   Navigate to the directory containing "mergen.py" in the terminal:
   
   $ python3 mergen.py --setup

   The wizard will guide you:
   > Database Location: Select your Syncthing folder (e.g., ~/Sync/mergen.db).
   > API Key: The entered key is stored encrypted.
   > Privacy Setting: Choose to enable or disable AI analysis.

B. MOBILE FIELD UNIT (ANDROID / TERMUX)
   Turn your phone into a Cyber Operations Device (Cyberdeck).
   
   1. Open Termux and install the necessary packages:
      $ pkg install python rust binutils build-essential clang
      $ pip install google-genai
   
   2. Navigate to your Syncthing folder and start the setup:
      $ cd /storage/emulated/0/Download/Mergen (Your specific path)
      $ python mergen.py --setup
   
   3. Paste this command AT ONCE to bypass Android restrictions:
      $ echo 'python /storage/emulated/0/Download/Mergen/mergen.py "$@"' > $PREFIX/bin/mergen && chmod +x $PREFIX/bin/mergen

[3] SECURITY AND PRIVACY (OPSEC)

1. SECURITY SHIELD v2.0
   - Mergen automatically masks Passwords (API_KEY=...) and IP Addresses
     (192.168.x.x) as (<GIZLI_IP_0>) before saving to the database.
   - This ensures no sensitive data leaks in screenshots or logs.

2. XSS & MEMORY PROTECTION
   - All inputs are filtered against HTML Injection attacks.
   - Large history files are processed line-by-line without bloating RAM.

3. ULTRA PRIVACY MODE
   - The AI module can be disabled during setup or via the GUI.
   - In this mode, not a single byte of data is sent to Google servers.

--------------------------------------------------------------------------------

[4] SYSTEM ARCHITECTURE
   - Decentralized: No central server. Your data is moved between devices via
     Syncthing with P2P encryption.
   - Hybrid Execution: Even when your computer is off, you can work on the local
     database using your phone's processor.
